# PM Frameworks & Playbooks

Quick reference for common frameworks. Ask Claude to walk you through any of these.

## Prioritization

### RICE Scoring

| Factor | Question | Score |
|--------|----------|-------|
| **R**each | How many users affected per quarter? | Number |
| **I**mpact | How much will it move the metric? | 0.25 (minimal) to 3 (massive) |
| **C**onfidence | How sure are we? | 100% / 80% / 50% |
| **E**ffort | Person-months required | Number |

**Score = (Reach × Impact × Confidence) / Effort**

### Value vs. Effort Matrix

```
High Value │ Quick Wins    │  Major Projects
           │ (Do First)    │  (Plan Carefully)
───────────┼───────────────┼──────────────────
Low Value  │ Fill-ins      │  Don't Do
           │ (Maybe)       │  (Avoid)
───────────┴───────────────┴──────────────────
             Low Effort      High Effort
```

### MoSCoW

- **Must have** - Non-negotiable requirements
- **Should have** - Important but not critical
- **Could have** - Nice to have if time permits
- **Won't have** - Explicitly out of scope (for now)

## Problem Definition

### 5 Whys

Start with the problem, ask "why" five times to find root cause.

1. Why? →
2. Why? →
3. Why? →
4. Why? →
5. Why? → Root cause

### Jobs to Be Done

> "When [situation], I want to [motivation], so I can [outcome]."

- **Situation**: Context/trigger
- **Motivation**: The "job" they're hiring the product to do
- **Outcome**: The progress they want to make

## Strategy

### Porter's Five Forces

1. **Competitive Rivalry** - Existing competition intensity
2. **Supplier Power** - Leverage of your suppliers
3. **Buyer Power** - Leverage of your customers
4. **Threat of Substitution** - Alternative solutions
5. **Threat of New Entry** - Barriers to entry

## Communication

### BLUF (Bottom Line Up Front)

Structure communications:
1. **Lead with the ask/decision needed**
2. Background context
3. Supporting details
4. Clear next steps

### RACI Matrix

| Task | Responsible | Accountable | Consulted | Informed |
|------|-------------|-------------|-----------|----------|
| | Does the work | Owns outcome | Input needed | Kept in loop |

## Stakeholder Management

### Stakeholder Map

```
High Power  │ Keep Satisfied │ Manage Closely
            │                │
────────────┼────────────────┼─────────────────
Low Power   │ Monitor        │ Keep Informed
            │                │
────────────┴────────────────┴─────────────────
              Low Interest     High Interest
```

### Influence Without Authority

1. Build relationships before you need them
2. Understand their goals and pressures
3. Find the win-win
4. Use data, not opinions
5. Get early buy-in on direction

## Meeting Frameworks

### DACI for Decisions

- **D**river - Runs the process, doesn't decide
- **A**pprover - Makes the final call
- **C**ontributors - Provide input
- **I**nformed - Need to know outcome

### Stand-up Format

1. What did you complete?
2. What are you working on?
3. Any blockers?

## Insurance-Specific

### Personal Lines Basics

- Auto, Home, Renters, Umbrella
- High volume, lower touch
- Regulatory compliance critical
- Renewal retention focus

### Commercial Lines Basics

- Business property, liability, workers comp, commercial auto
- Lower volume, higher complexity
- Relationship-driven
- Customized coverage structures

### Key Insurance Metrics

| Metric | What It Measures |
|--------|------------------|
| Loss Ratio | Claims paid / Premiums earned |
| Combined Ratio | (Losses + Expenses) / Premiums |
| Retention Rate | Policies renewed / Policies up for renewal |
| Quote-to-Bind | Quotes that become policies |
| Policy Lifetime Value | Total premium over relationship |
