Add Created column to action items schema

Track when each action item entered the list, separate from the
existing Date column (which is the source/origination date). Without
this, items that have been around for months blend in with fresh
ones, making it hard to spot stale candidates for archiving.

Schema (all three tables: Active, Completed, Archived):
- New Created column inserted right after Date
- Date stays as the source date (meeting, email, originating event)
- Created is the date the row was added to the list, populated on
  every new row and preserved across mark-complete and archive

Workflows:
- workflows/action-items.md documents the Date vs Created split and
  the rule that Created never changes once set
- workflows/process-transcript.md updates the append format so
  transcript-derived items use the meeting date for Date and today
  for Created
- templates/action-items.template.md picks up the new 8-column
  Active schema and 6-column Completed/Archived schemas

Artifact (templates/action-items-artifact.template.html):
- Each row shows an age line: colored pill (red for stale, amber
  for 14-29 days, gray for fresh) plus the added date
- New "Stale (>30d)" filter pill and Stale stat tile in the header
- Quick-add stamps Created with today's date
- mark-complete and archive ops preserve Created
- serializeMarkdown writes the new column on every auto-save

Build script (scripts/build-action-items-artifact.js):
- Parser reads Created from all three tables and threads it into
  the seed JSON
