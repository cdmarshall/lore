# STATE.md, session memory

> Gitignored personal file. Read at session start, updated before every substantive session ends (CLAUDE.md → Key behaviors → State file). Keep under ~150 lines: prune resolved items instead of appending forever. Facts graduate: an open failure becomes a verified fact once checked, then becomes a general rule if it applies beyond the specific case.

## Verified facts

Things checked against reality, not guessed. Stop re-deriving these.

- <fact, with how it was verified and the date>

## General rules

Distilled lessons that apply beyond one case. If a rule is universal to all Lore installs, promote it to the committed layer (a workflow, `_conventions.md`, or `VOICE.md`) and delete it here.

- <rule>

## Open failures

Things that went wrong, with the current hypothesis. Investigate before repeating the operation.

- <date>: <what failed> Hypothesis: <why>. Next: <check>.

## Lessons learned

Post-mortems worth keeping that are not yet rules.

- <date>: <lesson>

## Last session

One short block, overwritten each time: what happened, what was left open, what the next session should do first.

- <date>: <summary>. Next: <first move>.
